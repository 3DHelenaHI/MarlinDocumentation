G0, G1 - Linear Move

1.0.0-beta planner Add a straight line movement to the planner

The effect of G1 is to simply add a move to the queue, the move itself will happen at some point in the future. All the specified axes will move simultaneously to arrive at the given coordinates at the same time using linear interpolation. The speed may change over time following an acceleration curve, according to the acceleration and jerk settings of the given axes. Marlin treats G0 (rapid linear movement) as an alias to G1 (rapid movement). Some G-Code generators may, by convention, use G0 for non-extrusion movements (those without the E axis) and G1 for moves with the extruder.

Usage

G0 [E<pos in mm>] [F<mm/min>] [X<pos in mm>] [Y<pos in mm>] [Z<pos in mm>]

ArgumentDescription
[E<pos in mm>] The amount to extrude between the starting point and ending point
[F<mm/min>] The maximum movement rate of the move between the starting and ending point
[X<pos in mm>] A coordinate on the X axis
[Y<pos in mm>] A coordinate on the Y axis
[Z<pos in mm>] A coordinate on the Z axis

G2, G3 - Controlled Arc Move

1.0.0-beta planner Add a arc movement to the planner

G2 will make the tool draw a clockwise arc, G3 will make it do a counter-clockwise arc. Arcs are drawn as perfect circular segments, with the number of degrees (hence, length) determined by the angular difference between the start point and the end point with respect to the center point, by default G2/G3 will draw arcs with 1mm segments, so arcs below that threshold are not recommended.

Usage

G2 [E<pos in mm>] [F<mm/min>] <I<offset in mm>> <J<offset in mm>> [X<pos in mm>] [Y<pos in mm>]

ArgumentDescription
[E<pos in mm>] The amount to extrude between the starting point and ending point
[F<mm/min>] The maximum movement rate of the move between the starting and ending point
<I<offset in mm>> An offset from the current X position to use as the arc center
<J<offset in mm>> An offset from the current Y position to use as the arc center
[X<pos in mm>] A coordinate on the X axis
[Y<pos in mm>] A coordinate on the Y axis

Examples

Move in a clockwise arc from the current position to (125, 32) with the center offset from the current position by (10.5, 10.5).

Move in a counter-clockwise arc from the current position to (125, 32) with the center offset from the current position by (10.5, 10.5).

Move in a complete clockwise circle with the center offset from the current position by 20, 20.

G28 - Move to Origin

1.0.0-beta planner Home one or more axes

When the firmware receives this command, it moves all (or the supplied) axis’s back to the zero endstops as quickly as it can, then backs off by a [XYZ]_HOME_BUMP_MM and slowly moves back to the zero endstop activation points to increase position accuracy.

Usage

G28 [X] [Y] [Z]

ArgumentDescription
[X] Flag to go back to the X axis origin
[Y] Flag to go back to the Y axis origin
[Z] A coordinate on the Z axis

This process is also known as “Homing”.

G29 - Detailed Z-Probe

1.0.0-beta planner Probes the bed at 3 or more points

Probes the bed at 3 or more points and prints a report. The printer must be homed with G28 before G29.

Usage

G29 [B<pos in mm>] [D] [F<pos in mm>] [L<pos in mm>] [P<int>] [R<pos in mm>] [S<mm/min>] [T] [V<1|2|3|4>]

ArgumentDescription
[B<pos in mm>] Set the back limit of the probing grid
[D] Dry-Run mode, don't apply or clean the rotation Matrix
[F<pos in mm>] Set the front limit of the probing grid
[L<pos in mm>] Set the left limit of the probing grid
[P<int>] Set the size of the grid that will be probed (P x P points)
[R<pos in mm>] Set the right limit of the probing grid
[S<mm/min>] Set the XY travel speed between probe points
[T] Generate a Bed Topology Report
[V<1|2|3|4>] Set the verbose level

The command G29 can be used standalone to apply the Automatic Bed Leveling algorithm using the default values set in Configuration.h.

G4 - Dwell

1.0.0-beta planner Add a pause moment to the planner

Dwell pauses the command queue, and waits for a given period of time.

Usage

G4 [P<time in ms>] [S<time in sec>]

ArgumentDescription
[P<time in ms>] Amount of time to dwell
[S<time in sec>] Amount of time to dwell

If both S and P are included, S will take precedence.

Using G4 with no arguments is effectively the same as M400.

G5 - Bézier cubic spline

1.1.0-RC7 planner Cubic B-spline with XYZE destination and IJPQ offsets

G5 creates a cubic B-spline in the XY plane with the X and Y axes only. P and Q must both be specified for every G5 command. For the first G5 command in a series of G5 commands, I and J must both be specified. For subsequent G5 commands, either both I and J must be specified, or neither. If I and J are unspecified, the starting direction of this cubic will automatically match the ending direction of the previous cubic (as if I and J are the negation of the previous P and Q).

Usage

G5 <I<pos in mm>> <J<pos in mm>> <P<pos in mm>> <Q<pos in mm>> <X<pos in mm>> <Y<pos in mm>>

ArgumentDescription
<I<pos in mm>> X incremental offset from start point to first control point
<J<pos in mm>> X incremental offset from start point to first control point
<P<pos in mm>> X incremental offset from start point to first control point
<Q<pos in mm>> X incremental offset from start point to first control point
<X<pos in mm>> A coordinate on the X axis
<Y<pos in mm>> A coordinate on the Y axis

It is an error if an axis other than X or Y is specified.

G92 - Set Position

1.0.0-beta planner Set the current position of one or more axes

Allows programming of absolute zero point, by reseting the current position to the values specified.

Usage

G92 [E<pos in mm>] [X<pos in mm>] [Y<pos in mm>] [Z<pos in mm>]

ArgumentDescription
[E<pos in mm>] New extruder position
[X<pos in mm>] New X axis position
[Y<pos in mm>] New Y axis position
[Z<pos in mm>] New Z axis position

The command G92 will accept any value, coordinates outside the boundaries of the machine may cause undefined behavior.
